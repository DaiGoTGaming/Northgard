リポジトリ更新ルール

リポジトリ更新のタイミングとしては以下の2つがあります。

1.Northgardアップデート後、初めて日本語化パッチ更新する場合
2.Northgardは更新されていないが日本語化パッチを更新する場合

コミット時のサマリー(Summary)は以下のようにしてください。

対応するNorthgardのバージョン + (コミット時の年 + 月 + 日 + 時 + 分)

[例]
v3.3.7.35808(202312170855)

===========================================================
■Northgardアップデート後、初めて日本語化パッチ更新する場合
===========================================================

更新するファイルと更新内容は以下の通りです。(7ファイル)

Northgard
│  Northgard_jp_patch_for_mac.zip [export_zh.xmlとtexts_zh.xmlのみを差し替えて更新]
│  Northgard_jp_patch_for_windows.zip [export_zh.xmlとtexts_zh.xmlのみを差し替えて更新]
│  README.md [対応しているバージョン の部分とNorthgardの更新日付のみを修正して更新]
│
└─localize
    │
    ├─export
    │      export.xml [最新のNorthgardのもので更新]
    │      texts.xml [最新のNorthgardのもので更新]
    │
    └─import
       │
       └─extra
           │
           └─lang
                    export_zh.xml [修正後の日本語言語ファイルで更新]
                    texts_zh.xml [修正後の日本語言語ファイルで更新]

===========================================================
■Northgardは更新されていないが日本語化パッチを更新する場合
===========================================================

更新するファイルと更新内容は以下の通りです。(4ファイル)

Northgard
│  Northgard_jp_patch_for_mac.zip [export_zh.xmlとtexts_zh.xmlのみを差し替えて更新]
│  Northgard_jp_patch_for_windows.zip [export_zh.xmlとtexts_zh.xmlのみを差し替えて更新]
│
└─localize
    │
    └─import
       │
       └─extra
           │
           └─lang
                    export_zh.xml [修正後の日本語言語ファイルで更新]
                    texts_zh.xml [修正後の日本語言語ファイルで更新]

===========================================================
■注意点
===========================================================

上記ファイル以外のファイルは一切修正、削除など行わないでください。
日本語化パッチのZipファイル内のファイル構成には一切変更を加えないでください。

===========================================================
■実際の更新方法
===========================================================
GitHubリポジトリの更新にはGitHubのユーザー登録、及び DaiGoTGaming/Northgard の更新権限が必要です。
ご協力頂ける場合、X(@daigotgaming)までDMいただき、GitHubのユーザーメールアドレスを教えていただければ
更新できるようリポジトリにご招待します。

GitHubリポジトリを更新するには GitHub Desktop アプリを使用します。

・Download GitHub Desktop
https://desktop.github.com/

アプリをインストールしたら起動して、まずはNorthgardリポジトリを自身のPCにコピーします。
これにはGitHub Desktopのメニューの[File]->[Clone repository...]をクリックします。
Filter your repository のところに以下を入力します。

DaiGoTGaming/Northgard

Local path の右にある[Choose...]をクリックしてコピー先を選択します。
最後に[Clonoe]をクリックすると、GitHubのNorthgardリポジトリの内容が自身のPCにコピーされます。これをローカルリポジトリと呼びます。
今後、このローカルリポジトリに加えた変更は、GitHubのNorthgardリポジトリに変更を加えることになるということを念頭に置き、一切不必要な
修正は加えないようにしてください。(ファイルの変更のみならず、このフォルダ配下のファイルの削除、ファイルの追加もです)

ここから日本語化作業に入るわけですが、その前にかならずフェッチ(Fetch)、という作業を行います。
フェッチは、GitHubのNorthgardリポジトリの内容を取得します。
フェッチを行うには、GitHub Desktopの右上にある[Fetch origin]のボタンをクリックするだけです。
もしGitHubのNorthgardリポジトリに修正が加えられていた場合、[Pull origin]ボタンをクリックしてローカルリポジトリに統合します。
これを行わないと、別の誰かが修正した内容を自分の修正で上書きして削除してしまう可能性があります。

ここまで終わったら、export_zh.xmlとtexts_zh.xmlを修正して日本語化作業を行います。
ただ、念のため修正対象のファイルは、別途コピーして別の場所で修正作業を行いましょう。

修正が終わったら、このローカルリポジトリ内のファイルを更新します。
どのファイルを更新するかは、上で述べたとおりです。

その状態でGitHub Desktopを起動すると、更新されたファイルが一覧に表示されます。
念のためそれぞれのファイル比較をして、修正に問題ないかを確認します。
問題無ければコミットサマリーを入力して[Commit to main]をクリックします。
これによりローカルリポジトリの内容が更新されます。これはローカルのGitという管理システムに更新を記録した、ということです。

ただ、ここまでではローカルの内容が更新されたに過ぎません。
最後にGitHubのNorthgardリポジトリ(これをリモートリポジトリと呼びます)を更新する必要があります。
これにはプッシュ(Push)を行う必要があります。
右上にある[Push origin]のボタンをクリックするとPushが完了します。
これによりローカルリポジトリにおこなった修正がリモートリポジトリに反映され
すべての修正内容が反映された日本語化パッチが誰でもダウンロード可能となります。

リポジトリに加えた修正は、すべての人が自由に閲覧可能と言うことを忘れずに。
すべての修正は記録されるので、たとえ削除したとしても削除前の状態も見ることができるということを忘れずに。
※記録されるのはCommitのタイミングですので、CommitとCommitの間で追加->削除をおこなった場合は記録されません。
